# 📁📄 Product Dossier — система управления документацией изделий

**Product Dossier** — это настольное корпоративное приложение для централизованного хранения, просмотра, редактирования и управления документацией изделий. Система предназначена для автоматизации ведения «дела изделия», упрощения поиска нужных документов и повышения эффективности внутреннего документооборота.

Проект разрабатывался и развивался в рамках производственной практики в **АО «НИИ Телевидения»**, **научно-производственный комплекс №41**, с **16 февраля 2026 г.** по **28 марта 2026 г.**

---

## 📖 Обзор

Приложение предназначено для работы с изделиями и связанной с ними документацией.  
Основной сценарий работы строится вокруг иерархии:

**изделие → категория документов → документ → файл**

Система позволяет:

- централизованно хранить информацию об изделиях и документах
- быстро искать нужные документы по фильтрам
- управлять файлами и категориями документов
- контролировать историю изменений
- разграничивать права пользователей по ролям
- безопасно удалять и восстанавливать данные через корзину

## 🚀 Основные возможности

### 🔐 Пользователи и доступ
- Регистрация пользователей
- Авторизация с использованием учетных данных PostgreSQL
- Просмотр профиля текущего пользователя

- Добавление категорий документов

- Добавление дочерних документов
- Редактирование данных документа
- Изменение категории документа
- Открытие файлов в стандартных приложениях Windows
- Просмотр свойств файла

- Фильтрация по номеру или названию изделия

### 🗑️ Интегрированные модули
- **Корзина** для временного хранения удалённых изделий и документов
- Восстановление объектов из корзины
- Окончательное удаление объектов из корзины
- **Drag & Drop** загрузка документов в окна добавления документа и дочернего документа
- Автоматическое определение имени файла и автоподстановка названия документа
- **Смена пароля пользователя** с проверкой прав доступа и валидацией нового пароля

## 🧩 Ролевая модель

В приложении реализовано разграничение прав доступа по ролям:

- **Employee** — базовый пользователь
- **Admin** — расширенные права управления документами
- **SuperAdmin** — управление ролями, смена паролей, полный доступ к корзине

Особенности:
- только **Admin** и **SuperAdmin** имеют доступ к корзине
- только **SuperAdmin** может менять роли пользователей
- только **SuperAdmin** может менять пароли пользователей
- другой аккаунт с ролью **SuperAdmin** нельзя принудительно перевести на новый пароль

## 🗂️ Структура данных и хранения

Система работает не только с базой данных, но и с файловым хранилищем документов.

Поддерживается:
- хранение файлов изделий в основной папке
- работа с путями через конфигурационный файл
- использование сетевого пути хранения
- локальный резервный путь, если сетевой ресурс недоступен
- отдельное файловое хранилище для корзины

## 🌐 Развертывание

Проект ориентирован на использование в локальной сети предприятия.

Поддерживаемый сценарий развертывания:
- PostgreSQL разворачивается на отдельном серверном компьютере
- клиентское WPF-приложение устанавливается на рабочие станции пользователей
- приложение подключается к БД по сетевому адресу
- файловое хранилище может использовать сетевой диск с локальным резервным путём
